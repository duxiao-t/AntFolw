# agent.md

本文件是 Codex / Agent 在移动端 UI 整改任务里的上下文说明，与 `codex.md` 互补：后者是仓库级速查表，本文件是「当前任务」的输入约束、视觉基线与执行原则。

## 任务输入

- 设计基线：`.superpowers/brainstorm/mobile-design-20260718/content/all-key-pages-gallery.html`（18 个关键页面 + 1 个离线状态）。
- 风格方向：`.superpowers/brainstorm/mobile-design-20260718/content/enterprise-visual-directions.html` 中选定的「A. 清晰秩序」（Ant Design 企业蓝 + 中性背景）。
- 视觉约定：`mobile-data-flow.html` 给出的架构边界（feature 隔离、平台隔离、品牌隔离）。
- 用户诉求：现有实现与设计稿不一致，必须按设计稿重做，使 UI 与动画与设计稿「一模一样」。

## 视觉与排版原则

- 主色：`--af-color-primary: #1677ff`；成功 `#31a354`、警告 `#fa8c16`、危险 `#ff4d4f`。
- 背景：`#f5f6f8`，卡片 `#ffffff`，分隔线 `#edf0f2`，正文 `#202830`，副文 `#8b949e`。
- 圆角：surface 8–10px，control 7px，sheet 顶部 15px。
- 字体：标题 18–20px / 600–700，正文 11–13px，meta 9–10px；首选 `PingFang SC` / `Microsoft YaHei`。
- 底部导航固定 55–57px，三个 Tab 等宽，激活色用主色。
- 列表/卡片间距：卡片 8px，组间距 12px，行内 7–11px。
- 蓝色品牌欢迎卡：背景主色圆角 9–10px，内含「问候 + N 项待办」徽章；徽章是白底主色字。
- 应用宫格图标：33–38px 圆角 9–10px；奇数项主色蓝，偶数项绿（`#389e0d`）交替。
- 状态 tag：进行中 = 蓝底蓝字、已通过 = 绿底绿字、已驳回 = 红底红字、已完成 = 灰底灰字。
- 时间线：左边线 1px `#d9dde2`，节点圆点 7px（已完成主色 / 进行中白底橙边 / 未到达浅灰）。

## 关键页面骨架（按设计稿）

### 01 登录

- 蓝底圆角「A」方块 54×54px。
- 标题 28–30px / 800；副标题 13px 灰「移动审批，让每一次流转清晰可见」。
- 用户名 / 密码单边框 input，背景透明而非白卡。
- 全宽主色按钮「登录」。
- 底部 12px 灰字「企业微信免登录（二期）」。

### 02 工作台

- 顶部白底横条：左 `工作台 / AntFlow 科技`，右头像圆 29px。
- 蓝色品牌卡：`下午好，{name}` + `高效完成今天的工作` + `N 项待办` 徽章。
- 「常用应用」白卡：标题 + `全部应用 ›` 链接 + 4×2 应用宫格 + 第 8 项 `＋ 更多` 占位。
- 「最近流程」白卡：标题 + `查看全部 ›` 链接 + 行（左侧 7px 圆点 + 主标 + meta + 状态 tag）。

### 03 全部应用

- 顶部导航 `‹ 全部应用` + 右上 `管理` 链接。
- 搜索框（⌕ 搜索应用）。
- 分类 chip：全部 / 人事 / 财务 / 行政。
- 多个白卡分组：`最近使用` / `人事申请` / `行政申请`，每组内部 4×N 应用宫格。

### 04 管理常用应用

- 顶部导航 `‹ 管理常用应用` + 右上 `完成` 链接。
- 「已添加」白卡 + 拖动手柄/排序（首期可用上下箭头）。
- 「可添加」白卡 + 行（含 + 添加 tag）。
- 底部可省略固定按钮。

### 05 动态表单

- 顶部导航 `‹ 请假申请` + 右上 `草稿` 链接。
- 「请假信息」白卡：请假类型 / 开始时间 / 结束时间 / 申请原因。
- 「附件」白卡：dashed 上传区 `＋ 添加图片或文件`。
- 底部固定白底：`下一步` 主色按钮。

### 06 自选审批人

- 搜索框 `⌕ 搜索姓名`。
- 「直属主管 / 单选」白卡：人员行（头像 + 姓名 + 部门 + radio）。
- 「最近选择」白卡：人员行。
- 底部可省略。

### 07 提交确认

- 顶部导航 `‹ 确认提交`。
- 应用图标卡（图标 + 标题 + 副标题）。
- 「申请摘要 / 修改」白卡：请假类型 / 请假时间 / 请假时长 / 申请原因。
- 「审批流程」白卡：左侧时间线节点（已到达主色圆点 / 待办橙边白底 / 未到达浅灰）。

- 替换现有顶部横幅，保留恢复后自动重试。

## 动画与微交互

- 卡片、按钮、状态 tag 切换使用 120–200ms ease 过渡；避免长动画。
- 时间线节点初次渲染使用 fade-in + 上移 8px，按节点序号 stagger 50ms。
- 弹层 sheet 使用 `transform: translateY(100%) → 0`，250ms ease-out。
- 蓝色品牌卡可加 subtle hover lift（仅在 web 调试，不影响移动）。

## 工程约束

- `mobile/` 模块用 Vite + React 19 + antd-mobile v5 + TanStack Query + Zustand；不要引入 antd-mobile 之外的 UI 库。
- 不修改 `shared/api`、`shared/platform`、`shared/recovery`、`shared/telemetry`。
- 不修改 `auth.store`、`recoveryDraft.store`、`submitFlow.store`、`apps.store` 等核心 store 接口（仅允许新增派生 selector）。
- 所有改动必须通过 `npm run lint && npm test && npm run build`，并更新 `e2e/key-pages.visual.spec.ts-snapshots/*`。
- 视觉基线刷新走 `npm run test:e2e -- key-pages.visual.spec.ts`，需同步刷新四个视口（android-360 / iphone-375 / iphone-390 / iphone-430）。
